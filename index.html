Common Criteria (CC)

Basis for evaluation of security propertys of IT products; standard for IT security evaluation

Common set of requirements for security functionality and assurance measures (IT security requirements)

Permits comparability between results of independent security evaluations

Useful guide for: development, evaluation, procurement

shortcomings

  • assurance is limited to TOE

Quantitative Risk Analysis

ALE

SLE (Single loss expectancy) = AV(Asset value)*EF(Explosure factor)

ALE(Annualized loss expectancy) = SLE*ARO(Annual rate of occurence)

shortcommings

  • We cannot distinguish high-frequency, low-impact events from low frequency, high-impact events based on such a single simplified number

  • False sense of accuracy

Challenges
  • different strategies will have different cash flow
  • secondary effects, intangible assets
  • a lot of guess work

Actualizing risk analysis

Attack Trees

Qualify an attack in different dimensions

  • leaf nodes: attack initiation

  • root node: attack goal

  • and: conjuct to achieve the goal

  • or: different ways to achieve the goal

Attack Trees Can
  • be qualified with attributes(time, money, probability)
  • made as detailed as possible
  • identify the perferred attack vectors
Fault Tree Analysis (FTA)

FTA is a general purpose technique used for system reliability and safety engineering, risk assessment

  • an undesirable system state

  • arrived at using Boolean logic combining series of lower-level events

Event Tree Analysis (ETA)

Lays a path for accessing probability of various outcomes (in a single initiating event)

Failure Mode & Effect Analysis

Review as many components/subsystems as possible to identify failure modes, their causes and effects

RPN

Risk probability number (RPN) = prob * det * sev

  • prob (prob that the failure will occur)

  • det (prob that the failure will not be detected)

  • sev (damage that the failure mode may cause; severity)

Information Security

Cybersecurity & Infosec Differences
  • Infosec protects data across all mediums, Cybersecurity protects data in cyberspace
  • Cybersecurity protects non-information based assets
  • cybersecurity: Antivirus software, firewall
  • infosec: Encrypt data, providing security awareness training

Business Impact Analysis (BIA)

  1. identify critical things (products, processes, services) that need to be restore, meet its critical needs and statisfy regulations
  2. identify activities that support critical things
  3. assest impact overtime of not performing activities
  4. set prioritized timeframes (eg. maximus tolerable downtime)
  5. identify dependencies (includes 3-rd)

Business Continuity Planning (BCP)

  • Conduct BIA and RA (establish business recovery prorities, timescale&minimum requirements)
  • BC Strategy Formulation (options for meeting business recovery prorities, timescale&minimum requirements)
  • BCP Production (plans, organizations, detailed action task lists)
  • BCP Testing
  • BC Awareness
  • Ongoing BCP Maintenance

1-1.云使能技术包括哪些技术组件？用一句话简要说明其关键特征。

宽带网络和Internet架构 所有的云必须连接到网络
数据中心技术 利用数据中心技术集中存放IT资源
虚拟化技术 将物理IT资源转换为虚拟IT资源,从而提供了硬件无关性、服务器整合、资源复制、对资源更强的支持和可扩展性
Web技术 作为云服务的实现介值和管理接口
多租户技术 设计多租户技术使多个用户在逻辑上同时访问一个应用
服务技术 是云计算的基石，形成了“作为服务”的云交付模型的基础

​ 所有的云必须连接到网络，同时利用数据中心技术集中存放IT资源，应用虚拟化技术将物理IT资源转换为虚拟IT资源，从而提供了硬件无关性、服务器整合、资源复制、对资源更强的支持和可扩展性；使用Web技术作为云服务的实现介值和管理接口；设计多租户技术使多个用户在逻辑上同时访问一个应用；其中的服务技术是云计算的基石，形成了“作为服务”的云交付模型的基础。
